# cs201-final-project
yhf老师数算课期末总结！尝试给自己一个收束。
这个hub包含：
我机考用的cheatsheet，借鉴了乒乓球课认识的2025Spring的一个🌈大佬（phy24 ljh），我自己在此基础上加了一些调用的api示例，最终达到了看着cheatsheet就可以开始抄的程度。此外还增加了一些变式，比如根据字典序排序的拓扑排序，以及并查集的其他延伸等；
我这学期做过的题目单，整理的时候看着pycharm上好几下翻不到头的表真的很感慨；
Readme里面的一些面向cheatsheet编程小经验。

期末考试的教材书《从零开始构建大语言模型》，考题是从其自测书改编而来。我有些大意，只看了自测书，所以没选对那个BERT结构的题，虽然我把自测书仔仔细细的看了两三遍，但仍然有些小遗憾。
我把自测书进行了机翻，但机翻后的教材由于版权问题放上来仍然不太合适。
总的来说，我的数算学的不算特别好。我已经大三了，大一上学了王昭老师的C语言计概，成绩不错；但是对python几乎不了解。由于听说闫老师讲课给分都不错，自己也没什么别的机会了，就赶在秋季学期选了这个课。在上学期我选过信管系的《python数据分析》，对python语法虽说有了一定的了解，但是那个课作业和考试都允许用ai，所以我只掌握了读代码的能力，对于自己写python的能力是很欠缺的。
数算的学习是爬山般的。第一周的时候，我无法做到不问ai地写出最基本的语法；九月份课业任务相对小，我就坚持了一个月的每日选做。到九月末的时候，语法已经不是问题了。
但是算法和数据结构的要求仍然很高，对于stack、deque、heapq、tree、graph，我之前有的知道理论知识（毕竟工算专业必修离散数学），有的则完全没听说过。记得第三次机考的时候，出了一个对顶堆的题，我因为后面学习和科研的压力很大在中途有的放弃了数算的学习，在考试的时候一脸懵逼，考完后意识到自己落后了太多。
我觉得对我收获最大的，是十二月的学习。由于机考一天天挨过来了，我感受到前所未有的紧迫感。在花了一下午研究这个课的考试逻辑后，我惊喜地发现有些模板其实不需要会默写，只需要写在cheatsheet什么然后会抄会变式就行了。这给了我很大的希望，我先找到了coe25 lr同学（gy学长的身份给了一些幸运），问了问他觉得需要掌握的内容，然后用ljh同学的cheatsheet以及gpt进行了完善。
我发现，如果按照这种思路，光想着把基本算法写进去是不够的。你需要做很多题，大概了解这个模板的题会怎么出，有什么变式，然后把你一下子写不出来的多写几遍或者也把这个变式写在cheatsheet里面。
到距离考试还差一周的时候，我基本完成了基础知识的学习和cheatsheet的制作。接下来一周，我开始训练自己看着cheatsheet写题的能力，其实就是磨合这个“外挂”。我试着做了24和25Spring的题，竟然发现每次都能做出来4-5个。印象很深的是24Spring考了两个并查集，只要把并查集默写出来然后稍微改进一下就结束；25Spring的最小生成树和（看懂意思后的）神经网络其实也是默写题。对于树状dp，我确实不会，所以又吸纳进了cheatsheet。所以我在考试前的计划就是，拿下全部easy和medium，tough随缘。
上考场时候，遇到了很多意想不到的情况。必须承认今年的题比前两年都难，首先tough题不是那种会个模板就能做的，其次前两个题和T5都有难度。很幽默的是前两个题的正确做法我一开始不会，到最后也不会。二十分钟的徒劳无功之后，我先做了T3，模拟一下，注意到monster活着也算活着，秒了；T4这不拓扑排序吗，抄代码，抄完做一个dp，我成为了第六个作出T4的人。
T1T2仍然没有思路，我尝试先做T5。并查集是必须写的，写完之后，我先试了最基本的计算组内元素数，超时了；在这个时候，助教突然喊话第二题没有时间限制，绷不住的我对50000的数据写了个两重循环，waiting了十秒之后跳出一个蓝色的单词，我更绷不住了。然后我就想，T1暴力法可不可以？写了一个两重循环，又过了，我侥幸逃脱，这时候已经18:53。我又开始看T5，试了一些别的办法，但是最终没做出来。事后反思的时候，我感觉我的时间浪费了一部分在并查集的书写上面。我第一遍其实几乎打对了，但是有个变量名字打错了，这其实是因为我过于依赖自动补全导致最后也没查出来……然后就一直bug，我破釜沉舟删掉了重新打了一遍，然后当然还是遇到了这个问题，我静下心从头一行一行的对比代码，终于在最后一行发现了这个问题，但是已经过去了二十分钟。
总的来说，这次的考试是有遗憾但不遗恨的。做出4个已经是我期中的时候完全不敢想的，我当时甚至在犹豫要不要退课。但是如果没有波折，会不会能想到第五题的正确做法？明明很简单的。
从T1T2没卡时间能看出老师其实还是相当善良。这门课这次的机考区分度也很好，是一次不错的设计。（当然今年肯定是没有前面两次卷的哈哈哈哈）
如果有人问我闫老师的数算怎么样，我肯定会大力推荐。但是相比于其他好课的“好好听课”，我会建议他看两倍速的回放，建议他跟住每日选做，建议他按照老师的作业要求试着自己构建一个大模型并且好好看书而不是跳过第一周的这个题，建议他利用好这个工程式的上课风格，对自己的全面发展负责。
在大一下到大二的这段时间，我的同学们讨论起数算，我只能在一旁不语。现在我可以自信地说我掌握了它，至少是最基本的东西和语法，我可以试着把数据结构和算法用在计算几何等等更高深的内容上面，可以用python搓一些之前自己看着就害怕的代码，这就是没白上吧！
